C++ Код с Аргументами

October 25, 2023
У меня есть такое приложение #include <iostream> int main(int argc, char* argv[]) { std::cout << argv[1] << std::endl; return 0; }

У вас здесь представлен простой C++ программный код, который выводит на экран первый аргумент командной строки. Этот код ожидает, что при запуске программы будет передан хотя бы один аргумент командной строки, и он выводит первый аргумент на экран. Если аргументы не переданы, это может вызвать ошибку или непредсказуемое поведение.

Чтобы убедиться, что код работает правильно, вы можете выполнить его следующим образом:

  1. Сохраните код в файле с расширением ".cpp", например, "main.cpp".

  2. Затем скомпилируйте код в исполняемый файл. Вы можете использовать компилятор C++, такой как g++, следующим образом (если вы используете Linux):

    css
    g++ -o my_program main.cpp

    Это создаст исполняемый файл с именем "my_program".

  3. Запустите вашу программу, передав аргумент командной строки. Например:

    bash
    ./my_program Привет, мир!

    Результатом выполнения будет вывод "Привет, мир!" на экран.

Если вы запустите программу без аргументов командной строки, она может выдать ошибку, так как не будет предоставлен аргумент argv[1]. Поэтому убедитесь, что вы всегда предоставляете аргумент при запуске программы.

Как будет выглядеть скрипт для windows который запустить 5 экземпляров этого приложения и для каждого экземпляра будет передавать разный аргумент из массива

Для создания скрипта на Windows, который запустит 5 экземпляров вашей программы и передаст разные аргументы из массива, вы можете использовать пакетный файл (batch file). Вот пример такого скрипта:

batch
@echo off
setlocal

rem Массив аргументов
set "arguments[1]=Аргумент1"
set "arguments[2]=Аргумент2"
set "arguments[3]=Аргумент3"
set "arguments[4]=Аргумент4"
set "arguments[5]=Аргумент5"

rem Путь к исполняемому файлу вашей программы
set "executable=путь_к_вашей_программе.exe"

rem Запуск 5 экземпляров программы с разными аргументами
for /l %%i in (1,1,5) do (
  start "" "%executable%" !arguments[%%i]!
)

endlocal

В этом скрипте:

  1. Вы создаете массив аргументов arguments[1] до arguments[5], где каждый элемент массива представляет собой аргумент для вашей программы.

  2. Укажите путь к исполняемому файлу вашей программы в переменной executable.

  3. С помощью цикла for скрипт запускает 5 экземпляров программы с разными аргументами, используя оператор start. !arguments[%%i]! используется для получения значения аргумента из массива.

Просто сохраните этот скрипт в файле с расширением ".bat" и запустите его. Он запустит 5 экземпляров вашей программы, каждый с разным аргументом из массива.

Приложение выводит !arguments[1]! !arguments[2]! !arguments[3]! !arguments[4]! !arguments[5]!

Извините за путаницу. Проблема связана с тем, что в пакетных файлах переменные окружения не могут быть доступны внутри цикла for. Вместо этого, вы можете использовать массив переменных, чтобы хранить значения аргументов, и передавать их в программу. Вот исправленный скрипт:

batch
@echo off
setlocal enabledelayedexpansion

rem Массив аргументов
set "arguments[1]=Аргумент1"
set "arguments[2]=Аргумент2"
set "arguments[3]=Аргумент3"
set "arguments[4]=Аргумент4"
set "arguments[5]=Аргумент5"

rem Путь к исполняемому файлу вашей программы
set "executable=путь_к_вашей_программе.exe"

rem Запуск 5 экземпляров программы с разными аргументами
for /l %%i in (1,1,5) do (
  start "" "%executable%" !arguments[%%i]!
)

endlocal

В этой версии скрипта используется setlocal enabledelayedexpansion, чтобы включить отложенное расширение переменных, что позволяет использовать ! для обращения к элементам массива внутри цикла for.

Сохраните этот скрипт в файле с расширением ".bat" и запустите его. Теперь он должен правильно передавать аргументы в вашу программу.

А можно скрыть окна которые отрываются

Чтобы скрыть окна командной строки, которые открываются при запуске экземпляров вашей программы, вы можете использовать следующую команду start с флагом /B (запуск в фоновом режиме) и флагом /MIN (сворачивание окна):

batch
@echo off
setlocal enabledelayedexpansion

rem Массив аргументов
set "arguments[1]=Аргумент1"
set "arguments[2]=Аргумент2"
set "arguments[3]=Аргумент3"
set "arguments[4]=Аргумент4"
set "arguments[5]=Аргумент5"

rem Путь к исполняемому файлу вашей программы
set "executable=путь_к_вашей_программе.exe"

rem Запуск 5 экземпляров программы с разными аргументами (без окон командной строки)
for /l %%i in (1,1,5) do (
  start /B /MIN "" "%executable%" !arguments[%%i]!
)

endlocal

Теперь скрипт будет запускать экземпляры программы без отображения окон командной строки. Это позволит вам скрыть окна, в которых выполняется ваша программа.

А можно ли такой же скрипт но для linux &

Да, вы можете создать аналогичный скрипт для Linux, который будет запускать вашу программу в фоновом режиме и скрывать окна командной строки. В Linux для этого можно использовать символ & после команды, чтобы запустить процесс в фоновом режиме, а также команду nohup, чтобы отключить вывод в терминал. Вот пример скрипта для Linux:

bash
#!/bin/bash

# Массив аргументов
arguments=("Аргумент1" "Аргумент2" "Аргумент3" "Аргумент4" "Аргумент5")

# Путь к исполняемому файлу вашей программы
executable="./путь_к_вашей_программе"

# Запуск 5 экземпляров программы с разными аргументами (в фоновом режиме)
for arg in "${arguments[@]}"; do
  nohup "$executable" "$arg" > /dev/null 2>&1 &
done

В этом скрипте:

  • Массив arguments содержит ваши аргументы.

  • Путь к исполняемому файлу задан в переменной executable.

  • Затем с помощью цикла for мы перебираем элементы массива arguments, используя nohup для запуска вашей программы в фоновом режиме и перенаправляем вывод в /dev/null, чтобы скрыть вывод.

Сохраните этот скрипт в файле с расширением ".sh" (например, "run_programs.sh") и сделайте его исполняемым с помощью команды chmod +x run_programs.sh. Затем вы можете выполнить его с помощью ./run_programs.sh, и он запустит вашу программу в фоновом режиме без отображения окон командной строки.

Не скрывай вывод других

Если вы хотите, чтобы вывод программы не отображался в терминале, но в то же время скрыть окно командной строки, вы можете воспользоваться командой nohup и перенаправлением вывода в файл. Вот скрипт для Linux, который делает это:

bash
#!/bin/bash

# Массив аргументов
arguments=("Аргумент1" "Аргумент2" "Аргумент3" "Аргумент4" "Аргумент5")

# Путь к исполняемому файлу вашей программы
executable="./путь_к_вашей_программе"

# Папка для записи вывода
output_folder="./output"

# Создаем папку для вывода, если она не существует
mkdir -p "$output_folder"

# Запуск 5 экземпляров программы с разными аргументами (в фоновом режиме и вывод в файл)
for arg in "${arguments[@]}"; do
  nohup "$executable" "$arg" > "$output_folder/output_$arg.txt" 2>&1 &
done

В этом скрипте:

  • Массив arguments содержит ваши аргументы.

  • Путь к исполняемому файлу задан в переменной executable.

  • Создается папка output_folder, куда будет записываться вывод каждого экземпляра программы.

  • Затем с помощью цикла for мы перебираем элементы массива arguments, используя nohup для запуска вашей программы в фоновом режиме, а вывод перенаправляется в файл с именем, соответствующим аргументу.

Таким образом, вывод каждого экземпляра программы будет записан в соответствующий файл в папке output, и окна командной строки будут скрыты.

А если я хочу чтобы был вывод в консоле

Если вы хотите видеть вывод программы в консоли, но при этом скрыть окно командной строки при запуске из скрипта, вы можете воспользоваться командой start с опцией /B для запуска программы в фоновом режиме, а также опцией /WAIT, чтобы ждать завершения программы перед завершением скрипта. Вот пример скрипта для Windows:

batch
@echo off
setlocal enabledelayedexpansion

rem Массив аргументов
set "arguments[1]=Аргумент1"
set "arguments[2]=Аргумент2"
set "arguments[3]=Аргумент3"
set "arguments[4]=Аргумент4"
set "arguments[5]=Аргумент5"

rem Путь к исполняемому файлу вашей программы
set "executable=путь_к_вашей_программе.exe"

rem Запуск 5 экземпляров программы с разными аргументами (в фоновом режиме)
for /l %%i in (1,1,5) do (
  start /B /WAIT "" "%executable%" !arguments[%%i]!
)

endlocal

В этом скрипте:

  • Массив arguments содержит ваши аргументы.

  • Путь к исполняемому файлу задан в переменной executable.

  • С помощью цикла for мы перебираем элементы массива arguments, используя start с опциями /B и /WAIT, чтобы запустить программу в фоновом режиме и дождаться её завершения, но при этом окна командной строки будут скрыты.

Таким образом, вывод программы будет отображаться в консоли, но окна командной строки не будут видны при запуске из скрипта.

Этот скрипт выполняет аналогичные действия, как и ваш исходный скрипт для Windows, но адаптирован для среды Linux. С помощью этого скрипта, программа будет запускаться с разными аргументами, и вывод будет отображаться в терминале.
